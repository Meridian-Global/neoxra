# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in Neoxra, please **do not open a public GitHub issue**.

Instead, report it privately by email to:

- **support@neoxra.com**

Please include as much useful detail as you can, such as:

- a description of the issue
- steps to reproduce it
- affected files, endpoints, or components
- potential impact
- any suggested remediation if you have one

## What to Expect

We will try to:

- acknowledge receipt within **5 business days**
- review and triage the report as quickly as possible
- keep communication clear if we need more information
- fix validated issues in a timeframe that matches severity and project capacity

Because this is an early-stage project, response times may vary, but we will make a good-faith effort to investigate all legitimate reports.

## Disclosure Guidance

Please avoid public disclosure until we have had a reasonable opportunity to investigate and address the issue.

If a report is confirmed, we may coordinate a responsible disclosure timeline with the reporter.

## Supported Versions

This project is early-stage and does not currently maintain long-term support branches.

In general:

- the latest code on `main` is the most likely to receive security fixes
- older branches, forks, and outdated deployments may not be supported

## Scope

Security reports are especially helpful for issues involving:

- authentication or credential handling
- secrets exposure
- unsafe deployment or build behavior
- API vulnerabilities
- package supply-chain concerns
- data exposure or permission issues

Thank you for helping keep Neoxra and its users safer.
